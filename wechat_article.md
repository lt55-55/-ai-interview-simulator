# 我用 Claude 独立完成了一个 AI 面试官，然后它反过来面试了我

> 一个双非应届生的第一次独立 AI 项目实录

---

事情要从两周前说起。

那天我和 Claude 聊了三个小时。不是问技术问题，是让它给我做了一次从外到内的深度剖析——身高体重、感情经历、心理素质、学习习惯，全摊开了。最后它给了我一个结论：

**「你的问题是傲慢——你看一眼就知道一件事的结局，所以你不屑于执行。」**

这句话打穿了我。

我一直以为自己是「缺乏动力」。它告诉我，是「觉得不值得」。

所以当它说「做一个真实上线的项目，比你编十个简历项目都强」的时候，我接了这个挑战。我说：**A，来吧。**

---

## 从零到一，全靠对话

第一天下午。我没有打开 IDE，没有画架构图，没有写一行伪代码。

我就是继续和 Claude 对话。

「帮我设计项目结构。」「Prompt 模板这样写合理吗？」「出题模块和评估模块要不要分开？」

它给了我完整的架构方案。不是那种「你可以用 React + Node.js」的废话，而是具体的——Streamlit 做前端（因为我不会写 JS）、Anthropic SDK 做 AI 层、SQLite 做存储。每一层的技术选型都解释了为什么适合我的场景。

**我第一次体会到：和 AI 结对编程，不是它替我写代码，而是它帮我想清楚再写。**

以前我写项目，想三成就开始敲键盘，写到一半发现方向错了，推倒重来。这次不一样——Claude 逼着我把「你要做什么」「为什么这样做」「用户怎么用」这些问题在动键盘之前全部想清楚。

---

## 第一行代码，到第一个 bug

Streamlit 的界面很快搭起来了。简历输入框、岗位选择器、开始面试按钮。看着页面一点点成型，有种久违的爽感。

然后真正的考验来了——对接 API。

DeepSeek 的 API 兼容 Anthropic 协议，Claude Code 一直在用，我以为就是填个 key 的事。

点击「生成面试题」，转圈，转圈，报错：

```
'ThinkingBlock' object has no attribute 'text'
```

我的第一反应是：什么鬼。

第二反应是：这个 bug 我可能搞不定。

第三反应——我深呼吸了一下，开始看 traceback。报错行在 `question_generator.py`，我在取 `response.content[0].text`。DeepSeek 的 API 返回了一个 `ThinkingBlock` 对象放在 content 列表的第一个位置，它没有 `.text` 属性。

这暴露了我一个坏习惯：**写 API 集成代码的时候，我只考虑了 happy path。** 我默认了 API 返回的第一个 content 一定是文本。API 打了我的脸。

修复很简单——三行代码：

```python
# 修复前
content = response.content[0].text

# 修复后
content = "".join(b.text for b in response.content if b.type == "text")
```

但这件事让我意识到：AI 帮你写了 90% 的代码，剩下的 10% 是 AI 写不出来的——那些只有在你亲手踩坑时才会建立的判断力。**AI 能给你鱼，但「知道哪里有暗礁」这件事，只能靠你自己下水。**

---

## 推送到 GitHub，差点卡死在最后一公里

代码写完了，准备推送。我之前的 GitHub 账号从来没有推过代码——是的，学了四年计算机，GitHub 上没有一个自己的项目。

然后我们卡在了 HTTPS 认证上。Git 没有凭据管理器，HTTPS 推送要密码，输入了又不认。SSH 密钥没配过，.ssh 目录都不存在。

Claude 帮我生成 SSH 密钥的时候，还出了一堆乱码——我的 Windows 用户名是中文「兰添」，路径里有中文，好几个工具直接罢工。

折腾了快一个小时，终于把密钥配好，推到 GitHub。

当终端显示 `* [new branch] main -> main` 的那一刻，我盯着屏幕笑了。不是因为技术有多难，是因为**我终于把一个「差不多完成了」变成了「完成了」。**

---

## 项目活了，然后它开始问我问题

我把自己测试用的简历粘贴进去，选了「AI 应用开发工程师」，点击生成。

五道题，第一道是行为面试：

> *「请分享一个你在独立开发过程中遇到的最棘手的技术或产品问题，你是如何分析问题原因、制定解决方案并最终解决的？」*

我愣了一下。这道题不就是在问我**今天下午刚经历的 ThinkingBlock bug**？

第二道更狠：

> *「作为 AI 产品经理，你如何看待当前大语言模型在垂直场景落地的核心挑战？产品经理应如何平衡技术能力与用户需求？」*

这是我自己写的 Prompt 生成的题目，但我被问到的时候，脑子里突然有一个瞬间——我在被一个系统考问，而这个系统是我自己定义的。我写了出题的 System Prompt，我设计了评分维度，我搭建了评估报告的模板。但现在坐在被面试位置上的，是我。

**我做的项目，反过来在塑造我。**

这就是 AI 产品最迷人的地方——你创造它，它回应你，最后你从它身上看到自己没看到的东西。

---

## 感想：关于 AI、关于独立开发、关于我

第一，**AI 降低的不是技术门槛，是「从 0 到 0.5」的心理摩擦。** 以前独立做项目，最大的阻力不是不会写代码，是面对空白文件夹的恐惧。现在你有一个永远不会不耐烦的队友，你跟它说「我不知道怎么开始」，它会帮你把第一步拆出来。门槛还在——你要懂技术、要做决策、要踩坑、要 debug——但启动的摩擦力消失了。

第二，**独立开发的核心能力不是写代码，是「定义问题」。** 和 Claude 协作的过程中，我发现自己花时间最多的地方不是让它写代码，而是反复修正我对问题的描述：「我要解决的是求职者缺少真实面试练习机会的问题，不是做一个更好的 ChatGPT」。当你把问题定义清楚了，解决方案会自己浮现。定义不清楚，写再多代码都是跑偏。

第三，**傲慢确实是我的问题，但解决傲慢的方法不是「逼自己勤奋」，是「找一个你无法预测结局的事」。** 体重管理我知道结局（少吃多动 = 10 公斤消失），语言学习我知道结局（日积月累 = 从不会到会）。但 AI 项目不一样——成品出来的那一刻，它反问我两个面试题，然后让我重新审视了自己的职业方向。**当一个你看过剧透的观众遇上一部你预测不了结局的电影，你终于愿意坐下来看完了。**

---

## 写在最后

如果你和我一样——

双非本科，没有实习，技术深度在同届不算突出，总觉得自己「差一点什么」但又说不清差在哪里；

我的建议是：**不要再想了。做一个真实上线的项目，把它推到 GitHub，用自己做第一个用户。**

不是因为项目本身能帮你拿 offer。是因为在这个过程中，你会被迫面对那些「想」发现不了的问题——API 返回格式变了怎么办、认证配置怎么搞、404 页面要不要做、README 写什么才能让别人一眼看懂你在做什么。

**这些问题不性感，但它们就是「脚踏实地的尝试」，是你我的傲慢最需要的解药。**
